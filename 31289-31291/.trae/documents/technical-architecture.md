# 高尔夫球模拟小游戏 - 技术架构文档

## 1. 技术选型

### 1.1 核心技术栈
- **HTML5**：页面结构和语义化标签
- **CSS3**：样式设计、动画效果、响应式布局
- **JavaScript (ES6+)**：游戏逻辑和交互控制
- **Canvas 2D API**：游戏画面渲染

### 1.2 无外部依赖
- 不使用任何第三方库或框架
- 所有功能原生实现，确保轻量和性能

## 3. 系统架构

### 3.1 模块划分

#### 3.1.1 游戏主控制模块 (game.js)
- **职责**：游戏状态管理、用户输入处理、游戏流程控制
- **主要功能**：
  - 初始化游戏
  - 管理游戏状态（瞄准、飞行、滚动、完成）
  - 处理鼠标事件（拖拽击球）
  - 协调各模块协作
  - 计分系统

#### 3.1.2 物理引擎模块 (physics.js)
- **职责**：模拟高尔夫球的运动物理
- **主要功能**：
  - 抛物线飞行轨迹计算
  - 滚动阻力和坡度影响
  - 碰撞检测与响应
  - 入洞判定

#### 3.1.3 球场数据模块 (course.js)
- **职责**：18洞球场数据定义
- **主要功能**：
  - 每洞的地形数据（高度图）
  - 发球台、球洞位置
  - 障碍物（沙坑、水池、树木）
  - 标准杆设定

#### 3.1.4 渲染引擎模块 (renderer.js)
- **职责**：Canvas画面绘制
- **主要功能**：
  - 球场渲染（草地、沙坑、水池）
  - 高尔夫球绘制
  - 方向和力度指示器
  - 计分板UI

### 3.2 数据流

```
用户输入 → 游戏控制 → 物理引擎 → 渲染引擎
                ↓
           计分系统
```

## 4. 核心数据结构

### 4.1 球对象
```javascript
ball = {
  x: number,           // X坐标
  y: number,           // Y坐标
  vx: number,          // X方向速度
  vy: number,          // Y方向速度
  isFlying: boolean,   // 是否在空中
```

```javascript
hole = {

  obstacles: array,         // 障碍物列表

}
```

### 4.3 游戏状态
```javascript
gameState = {
  currentHole: number,      // 当前洞数 (0-17)

  aimStart: { x, y },       // 瞄准起点
  aimEnd: { x, y }          // 瞄准终点
```

## 5. 核心算法

### 5.1 抛物线飞行算法
- 初速度：由拖拽距离决定
- 发射角度：默认45度（可通过拖拽方向微调）
- 重力加速度：模拟真实重力
- 高度公式：`h(t) = h0 + v0 * t - 0.5 * g * t²`

### 5.2 滚动物理算法
- 滚动阻力：草地阻力系数
- 坡度影响：根据地形高度差计算加速度
- 摩擦力：速度衰减
- 停止条件：速度小于阈值

### 5.3 碰撞检测
- 圆形碰撞检测（球与障碍物）
- 边界检测（球场边界）
- 入洞判定（球与球洞距离小于阈值）

## 6. 性能优化

### 6.1 渲染优化
- 使用requestAnimationFrame进行动画循环
- 按需重绘，避免全画面重绘
- 离屏Canvas预渲染静态元素

### 6.2 物理优化
- 固定时间步长物理更新
- 简化碰撞检测算法
- 合理设置物理参数，避免极端计算

## 7. 兼容性策略

### 7.1 浏览器兼容
- 使用ES6+语法，现代浏览器原生支持
- Canvas 2D API广泛支持
- CSS3特性添加浏览器前缀（如需）

### 7.2 响应式设计
- 使用百分比和vmin/vw单位
- Canvas自适应容器大小
- 布局使用Flexbox和Grid
